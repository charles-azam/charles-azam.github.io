# Projets

Faire fonctionner l'IA pour des problèmes d'ingénierie et techniques complexes.

---

## Jimmy Energy
**DSI (2022-2025)** • Transformation Engineering-as-Code

Transformation d'une entreprise d'ingénierie traditionnelle vers un workflow Git. Remplacement du PLM legacy par des outils Python personnalisés (PyJimmy). Toute l'équipe d'ingénierie travaille désormais sur une base de code unifiée avec contrôle de version, CI/CD et intégration IA.

**Impact:** Les ingénieurs passent leur temps à faire de l'ingénierie au lieu de gérer des fichiers. Données propres et versionnées permettant des workflows IA.

**Stack:** Python, Git, AWS, GitHub Actions

## OmniAgents
[GitHub](https://github.com/charles-azam/OmniAgents) • **Développement actif**

Interface unifiée pour les agents IA de codage à travers plusieurs environnements d'exécution (Local, Docker, E2B) et frameworks (smolagents, Pydantic-AI, LangChain).

**Problème:** L'infrastructure des agents IA est fragmentée. Chaque combinaison environnement/framework nécessite un code différent.

**Solution:** Abstraction complète. Écrire la logique une fois, exécuter partout.

**Stack:** Python, Docker, E2B

## Predibench
[Plateforme](https://predibench.com) • [GitHub](https://github.com/charles-azam/predibench) • [Article](https://huggingface.co/blog/charles-azam/predibench)

Évaluation des modèles IA sur des marchés de prédiction réels. Plateforme live testant si l'IA peut battre les humains en prévision.

**Pourquoi c'est important:** La plupart des benchmarks utilisent des datasets statiques. Les marchés de prédiction offrent une évaluation en temps réel avec des enjeux réels.

**Stack:** Python, Polymarket API, RAG
